Documentación Arquitectónica: La Cocina

Este documento interactivo consolida el modelado completo del módulo de cocina. El sistema gestiona el ciclo de vida de los pedidos, validando cruces de restricciones (existencia de stock vs. menú del día) y proveyendo un motor de consultas temporales para reportería (diaria, mensual, anual).

03 Diagrama Lógico Relacional (DLR)

Traducción del DER a esquema de tablas, definiendo PKs (Claves Primarias) y FKs (Claves Foráneas).

classDiagram class Pedidos { <<Table>> +PK id_pedido : INT +fecha : DATE +hora : TIME +estado : VARCHAR } class Detalle_Pedidos { <<Table>> +PK id_detalle : INT +FK id_pedido : INT +FK id_plato : INT +cantidad : INT } class Platos { <<Table>> +PK id_plato : INT +nombre : VARCHAR +precio : DECIMAL } class Menu_Diario { <<Table>> +PK FK id_plato : INT +PK fecha : DATE } class Materia_Prima { <<Table>> +PK id_mp : INT +nombre : VARCHAR +stock_actual : DECIMAL } class Recetas { <<Table>> +PK FK id_plato : INT +PK FK id_mp : INT +cantidad_necesaria : DECIMAL } Pedidos "1" -- "*" Detalle_Pedidos : id_pedido Platos "1" -- "*" Detalle_Pedidos : id_plato Platos "1" -- "*" Menu_Diario : id_plato Platos "1" -- "*" Recetas : id_plato Materia_Prima "1" -- "*" Recetas : id_mp

04 Diagrama de Transición de Estados (DTE)

Máquina de estados estricta que gobierna el ciclo de vida de un PEDIDO.

stateDiagram-v2 [*] --> PEDIDO : Se ingresa comanda al sistema state validacion_condicional <> PEDIDO --> validacion_condicional : Cocina recibe y evalúa validacion_condicional --> EN_PROCESO : [Stock == OK] AND [Plato IN Menu_Dia] validacion_condicional --> CANCELADO : [Stock == LOW] OR [Plato NOT IN Menu_Dia] EN_PROCESO --> TERMINADO : Preparación completada EN_PROCESO --> CANCELADO : Merma/Error de cocina insalvable TERMINADO --> [*] : Se entrega al mozo CANCELADO --> [*] : Se notifica al salón
